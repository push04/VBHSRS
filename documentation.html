1. Project Overview

The High Speed Rolling Stock (HSRS) Analysis Platform is a browser-based, real-time physics simulation and engineering analysis tool for high-speed passenger rail systems. It models traction, braking, wear, vibration, thermal behaviour, and ride comfort — entirely in-browser with no server dependency.

The platform was developed to support engineering analysis of high-speed rail (HSR) operations in India, particularly the transition from existing semi-HSR stock (160 km/h) to dedicated HSR rolling stock capable of 320 km/h on the Mumbai–Ahmedabad High Speed Rail corridor (508 km, MAHSR).

What Does It Simulate?

  • Real-time simulation loop — physics updated every 100 ms using Runge-Kutta 4th order (RK4) integration
  • Traction & braking — tractive effort curves, adhesion limits, braking distance
  • Rail-wheel tribology — contact stress, creep forces, wear volume (Hertz + Polach + Archard)
  • Suspension dynamics — frequency response, Sperling ride index, ISO 2631-1
  • Vibration analysis — vertical, lateral, jerk, yaw, roll with live charts
  • Thermal models — brake disc temperature, bearing temperature, hotbox detection
  • Energy consumption — Davis resistance, specific energy at speed
  • MAHSR corridor — ATC speed profiles, station geometry, journey time analysis

2. Purpose, Need & Benefits

Why This Platform Exists

Traditional rail engineering analysis requires expensive commercial software (SIMPACK, VAMPIRE, ADAMS/Rail) that demands specialist training and controlled environments. Engineering decisions about fleet configuration, track quality requirements, maintenance intervals, and speed upgrades are often made without accessible real-time sensitivity analysis.

This platform fills that gap by providing:

  • Instant parameter sensitivity — engineers can slide a parameter and see how braking distance, wear rate, bearing temperature, and ride quality change immediately
  • Comparative analysis — three reference datasets (Live, HSRS Baseline 160 km/h, MAHSR Target 320 km/h) visible simultaneously on all charts
  • Educational tool — formulae, standards compliance thresholds, and physical reasoning are embedded in the UI
  • No installation required — runs in any modern browser, shareable as a single HTML file

Engineering Benefits

  • Rapid assessment of speed upgrade implications (160 → 200 → 320 km/h)
  • Formation optimisation (10-car vs 16-car — tractive effort, energy, wear)
  • Track quality requirements for speed targets (Sperling index < 2.5 threshold)
  • Brake system sizing — stopping distance, disc temperature limits
  • Bearing maintenance interval planning (EN 12082 hotbox thresholds)
  • Conicity/wheel profile impact on hunting stability (Klingel wavelength)
  • MAHSR journey time estimation and ATC profile analysis

Operational Need

India's first HSR project (MAHSR, BKC–Sabarmati, 508 km, 320 km/h) requires engineering analysis of rolling stock performance that differs fundamentally from existing networks. At 320 km/h:

  • Aerodynamic drag dominates (~v³), making it 8× higher than at 160 km/h
  • Contact stress and wear rates escalate non-linearly
  • Hunting instability requires tight conicity control (< 0.05)
  • Bearing thermal loads require continuous monitoring (EN 12082)
  • Brake disc temperatures during emergency stops can exceed 400°C

3. Software Architecture

Technology Stack

  • Three.js r128 — 3D train body / bogie visualisation with OrbitControls
  • Chart.js 3.9.1 — all 2D physics charts (animation disabled for real-time streaming)
  • Pure JavaScript — no framework; IIFE module pattern for scope isolation
  • Single HTML file — all HTML, CSS, and JS in one deployable file (~6200 lines)

Simulation Loop

The main loop runs at 100 ms intervals (10 Hz). Each tick:

  1. User inputs read from sliders → physics constants computed (computeDynamics())
  2. RK4 integration over Δt = 0.1 s → new velocity, position
  3. Brake disc temperature updated (thermal ODE)
  4. All live charts updated (Chart.js update('none') — no animation)
  5. Hook chain fired → vibration IIFE receives same dyn object → bearing temp, vibration metrics updated

Module Hook System

The main simulation loop exposes window._lapUpdateHook(dyn, now). Secondary modules (LAP Panel, Vibration IIFE) chain onto this hook so they receive live physics data without coupling to the main loop's internals.

5. Physics Engine — Core Formulae

5.1 Resistance (Davis Formula)

Davis resistance describes the total running resistance of a train as a function of speed. It captures rolling resistance, bearing drag, and aerodynamic drag.
R_total = A + B·v [N/kN or N] where (for Indian broad gauge EMU/HSR): A = 1.5 + 0.01·W [N/kN] — rolling & bearing resistance B = 0.0065·v — aerodynamic drag coefficient term v = speed [km/h] Full resistance force [N]: F_resist = (A + B·v) × W_total [kN] / 1000 + W_total × g × sin(θ_gradient) + (aerodynamic: k_aero × v²) k_aero = 0.5 × ρ_air × C_D × A_front = 0.5 × 1.225 × 0.25 × 9.0 ≈ 1.38 [N/(m/s)²]

Why Davis?

Davis (1926) is the empirically validated standard for rail vehicle resistance. The A term captures speed-independent losses (wheel deformation, bearing seals); the B·v term captures aerodynamic skin friction and form drag. At 320 km/h the aerodynamic term dominates — it is ~4× the rolling resistance.

Effect of changing parameters on resistance:

  • Mass ↑ → A increases (heavier wheel loads), total resistance force increases proportionally
  • Speed ↑ → B·v term increases as v², aero as v³ → resistance grows rapidly
  • C_D ↑ (blunter nose) → aerodynamic resistance increases; explains why Shinkansen has long nose cones
  • Frontal area ↑ → proportional increase in aero resistance

5.2 Tractive Effort Model

Available TE [N]: TE_installed = (n_motors × P_motor) / v [at speed v m/s] TE_adhesion = µ_CK × W_powered [adhesion limit] TE_available = min(TE_installed, TE_adhesion) Net acceleration: a = (TE_available - F_resist) / M_total [m/s²] RK4 integration over dt = 0.1 s: v_new = v + (k1 + 2k2 + 2k3 + k4) / 6 where k1 = a(v), k2 = a(v + k1·dt/2), etc.

Why RK4?

RK4 is 4th-order accurate: error scales as O(dt⁵) vs O(dt²) for Euler. At 10 Hz update rate (dt = 0.1 s), RK4 gives velocity accuracy better than 0.01 m/s. The additional computational cost (4 function evaluations per step) is negligible in JS.

5.3 Curtius-Kniffler Adhesion

µ_CK(v) = µ_base × [0.161 + 7.5 / (v_kmh + 44)] where: µ_base = 0.33 (dry rail, clean) µ_base = 0.20 (wet rail) µ_base = 0.12 (contaminated / leaf) At 160 km/h (dry): µ = 0.33 × [0.161 + 7.5/204] ≈ 0.165 At 320 km/h (dry): µ = 0.33 × [0.161 + 7.5/364] ≈ 0.121 Adhesion force limit: F_adh = µ_CK × W_powered_axles [N]

Why Curtius-Kniffler?

Curtius-Kniffler (1950) is the UIC-standardised empirical adhesion model validated for European and Japanese HSR. It correctly captures the speed-dependency of wheel-rail adhesion — at higher speeds, the contact time decreases, moisture film cannot drain, and the real contact area (asperity interlocking) reduces. At 320 km/h, available adhesion is ~30% lower than at standstill.

Effect of changing parameters:

  • Speed ↑ → µ decreases → adhesion limit decreases → TE_max falls → longer acceleration from high speeds
  • Rail condition → wet → µ_base drops from 0.33 to 0.20 → 40% reduction in adhesion → risk of wheel-slip at lower TE
  • Motor ratio ↓ → fewer powered axles → W_powered ↓ → F_adh ↓ → TE limited earlier

6. Vibration Analysis

6.1 Klingel Hunting (Lateral Stability)

Klingel hunting wavelength: λ_k = 2π × √(R_wheel × e_gauge / λ_conicity) where: R_wheel = wheel radius [m] (= wheel_diam/2) e_gauge = half track gauge = 0.7175 m (1435 mm gauge) λ_conicity = wheel conicity (taper angle) Klingel frequency at speed v: f_k = v / λ_k [Hz] Critical hunting speed: Vc ≈ λ_k × √(k_lateral / M_bogie) / (2π) (simplified — depends on bogie yaw stiffness) In the platform: Vc estimated from Klingel with bogie mass 3000 kg, k_lat = 2.5 MN/m

Why Does Conicity Matter?

Wheel conicity is the primary design parameter controlling hunting stability. Higher conicity (worn wheel profile) produces shorter Klingel wavelength → lower critical speed Vc. When train speed approaches Vc, the bogie enters sustained lateral oscillation (hunting), causing:

  • Increased lateral acceleration (passenger discomfort)
  • Flange contact → rail gauge face wear → corrugation risk
  • Derailment risk if conicity is very high and speed is near or above Vc

HSR rule of thumb: maintain conicity < 0.05 for 320 km/h operation. Wheel reprofiling required when conicity exceeds 0.1.

6.2 Vertical Ride (ISO 8608 + ISO 2631-1)

Track irregularity PSD (ISO 8608): S_z(Ω) = A_r / Ω² [m² / (rad/m)] where Ω = spatial frequency [rad/m], A_r is roughness: TQ1 (excellent HSR): A_r = 0.5 × 10⁻⁶ TQ2 (good mainline): A_r = 2.0 × 10⁻⁶ TQ3 (average): A_r = 5.0 × 10⁻⁶ TQ4 (poor): A_r = 15.0 × 10⁻⁶ 1-DOF secondary suspension transmissibility: H(ω) = √[(1 + (2ζr)²) / ((1-r²)² + (2ζr)²)] where r = ω/ωn, ωn = √(k_s/m_car), ζ = damping ratio Vertical RMS acceleration: a_z,rms = √[ ∫ S_z(Ω) · H²(ω) · ω⁴ · dΩ ] × calibration_factor Integration bounds: Ω equivalent to 0.3–30 Hz at speed v calibration_factor = 0.00045 (matched to EN 12299 field data)

Parameter Effects on Vertical Vibration:

  • Track quality TQ↑ → A_r increases → a_z,rms increases proportionally → Sperling Wz rises → ride quality worsens
  • Speed ↑ → spatial frequency maps to higher temporal frequency → more of PSD falls in sensitive frequency range → a_z,rms typically increases
  • Spring stiffness k_s ↑ → ωn increases → resonance at higher frequency → less isolation in 1–10 Hz range → harder ride
  • Damping ζ ↑ → resonance peak suppressed → smoother ride near ωn; slight increase in high-frequency transmission

6.3 Sperling Ride Index (Wz)

Sperling Wz index: Wz = 4.42 × (a_z,rms)^0.3 × F(f) where F(f) is ISO 2631-1 frequency weighting: F(f) = 0.588 at 4–8 Hz (most sensitive vertical) F(f) = 1.0 at 8 Hz F(f) = 8/f above 8 Hz Simplified platform version: Wz ≈ 4.42 × (a_z_weighted)^0.3 Interpretation: Wz ≤ 2.5 — Excellent (imperceptible motion) 2.5–3.0 — Good (just perceptible) 3.0–3.5 — Acceptable (clearly perceptible) 3.5–4.5 — Poor (strongly perceptible) > 4.5 — Very poor (dangerous)

6.4 Jerk

Longitudinal jerk: J_base = |Δa / Δt| [m/s³] (acceleration rate of change) Gear mesh contribution: J_gear = 0.02 × |a| × f_gear (gear frequency oscillation) f_gear = 2π × (v/r_wheel) × (1/gear_ratio) / (2π) ≈ 10–40 Hz simplified Coupler slack: J_coupler = 0.15 × |a| × (1 - v/v_max) (higher at low speed) Total: J_total = J_base + J_gear + J_coupler [m/s³] Comfort limits (EN 13715): J < 1.0 m/s³ — Comfortable J < 2.0 m/s³ — Acceptable J > 2.0 m/s³ — Uncomfortable

6.5 Body Roll

Body roll angle from lateral acceleration: φ_roll = arctan(ay × M_car × h_cg / k_roll) where: ay = lateral acceleration [m/s²] h_cg = 1.5 m (centre of gravity height above roll centre) k_roll = 500,000 Nm/rad (anti-roll bar + air spring stiffness) M_car = car mass [kg] Roll rate = Δφ/Δt [deg/s] UIC limits: φ < 6° — acceptable body roll φ > 8° — exceeds limit; may require curve speed reduction

6.6 Bogie Yaw Rate

Klingel bogie yaw oscillation: ψ̇ = A_yaw × 2π × f_k [deg/s] where f_k = v/λ_k (Klingel frequency), and: A_yaw = 0.001 m (lateral displacement amplitude, normal tracking) Near hunting (v/Vc > 0.7): A_yaw grows as: 0.001 + 0.005 × max(0, v/Vc - 0.7) / 0.3 Comfort threshold: ψ̇ < 5 deg/s (EN 14363)

7. Thermal Models

7.1 Brake Disc Temperature

Brake disc thermal ODE: dT/dt = (Q_in - Q_cool) / (m_disc × c_p) Heat input during braking: Q_in = F_brake × v [W] (braking power) F_brake = µ_brake × N_normal × n_discs = brake_ratio × F_adhesion_limit × efficiency Cooling (convective, both faces): Q_cool = h_conv × A_disc × (T - T_ambient) h_conv = 15 + 1.5 × v_ms [W/m²K] (forced convection at speed v) A_disc = 0.2 m² (effective cooling area per disc) m_disc = 15 kg (cast iron disc mass) c_p = 500 J/kgK (cast iron specific heat) Ambient temperature T_ambient = 30°C (Indian summer baseline)

Why Do Discs Get So Hot?

Braking power Q_in = F × v is very large at high speed. For an emergency stop from 160 km/h (44.4 m/s) with a 16-car train (624 t): F_brake ≈ 400 kN → Q_in ≈ 17.8 MW split across 64 discs → 278 kW per disc. This raises temperature at ~278,000 / (15 × 500) = 37°C/s. Even a 10-second high-speed braking phase raises temperature by ~370°C.

Natural Cooling

At rest (v=0): h_conv = 15 W/m²K, Q_cool = 15 × 0.2 × (T-30) = 3(T-30) W. From 400°C: dT/dt = -3×370/(15×500) = -0.15°C/s → 41 minutes to cool from 400°C to 50°C. This is realistic for cast iron brake discs — they cool slowly at rest, which is why the Reset button exists for fresh test scenarios.

Parameter Effects:

  • Speed ↑ → Q_in increases proportionally; cooling increases too (h_conv ↑) but Q_in grows faster → higher peak temperature
  • Brake ratio ↑ → F_brake ↑ → Q_in ↑ → higher disc temp
  • Emergency mode → maximum F_brake → highest disc temperature (can exceed 500°C from 320 km/h)
  • More discs (n_discs ↑) → heat shared across more discs → lower temperature per disc

7.2 Bearing Temperature (EN 12082)

Bearing heat generation: P_bearing = µ_bearing × F_axle × v [W] where: µ_bearing = 0.0001 (rolling bearing overall friction incl. seals, lubrication drag) F_axle = W_total / (n_cars × 4 × n_axles_per_bogie × 2) per bearing ≈ total weight / total number of bearings v = speed [m/s] Effective thermal resistance: R_eff = 0.085 / (1 + 0.001 × v) [K/W] Steady-state bearing temperature: T_bearing = T_ambient + P_bearing × R_eff = 30 + µ_bearing × F_axle × v × R_eff Typical values: 160 km/h: ΔT ≈ 18–22°C above ambient → T ≈ 48–52°C (Normal) 320 km/h: ΔT ≈ 30–40°C above ambient → T ≈ 60–70°C (Normal/Watch) EN 12082 Hotbox Thresholds: Alarm: ΔT > 50K OR T > 80°C Danger: ΔT > 80K OR T > 110°C

Why µ_bearing = 0.0001?

A precision CARB/TBU (tapered roller) bearing for axleboxes has rolling contact friction µ ≈ 0.0005–0.001. However, the effective coefficient including viscous drag from grease, seal friction, and gear side loads, at operating temperatures where grease is fully fluid, falls to µ_eff ≈ 0.0001–0.0003. The value 0.0001 corresponds to a well-maintained bearing at operating temperature — conservative but realistic for simulation purposes. It correctly produces ΔT ≈ 20°C at 160 km/h, consistent with EN 12082 normal class.

8. Rail-Wheel Tribology

8.1 Hertz Contact Theory

Hertz elliptical contact: Contact half-axes a, b: a = m_H × ∛(3F·R_eff / (2E*)) [m] b = n_H × ∛(3F·R_eff / (2E*)) [m] m_H, n_H = Timoshenko coefficients (function of curvature ratio e*) e* = (R_w - R_r) / (R_w + R_r) Reduced elastic modulus: 1/E* = (1-ν_w²)/E_w + (1-ν_r²)/E_r For steel/steel: E* ≈ 113.6 GPa Effective radius: R_eff = R_wheel × R_rail / (R_wheel + R_rail) R_rail = 0.30 m (rail head crown radius, UIC60) Peak Hertz pressure: p₀ = 3F / (2π × a × b) [Pa] Contact area: A_contact = π × a × b [m²] Normal force per wheel: F = W_total / (n_axles × 2) [N]

Why Hertz Contact?

Hertz theory is the closed-form solution for elastic contact between curved bodies. It is the foundation of all rail-wheel contact mechanics. Peak pressure p₀ determines whether material yields (p₀ vs shakedown limit ≈ 2.8 × yield strength = 2.8 × 900 MPa ≈ 2.5 GPa for R260 rail steel). Below shakedown, only elastic deformation occurs. Above it, ratchetting wear accelerates.

8.2 Polach Creep Model

Polach (2000) tangential contact force: F_T = (2µN/π) × [ε/(1+ε²) + arctan(ε)] where: ε = (π × C_11 × ξ × a × b) / (4 × µ × N) ξ = creep ratio (normalised slip velocity) C_11 = Kalker longitudinal creep coefficient ≈ 10–12 GPa/m µ = available adhesion (Curtius-Kniffler) N = normal force [N] Longitudinal creep ratio: ξ_x = (v_wheel - v_rail) / v_rail (slip fraction) Lateral creep from hunting: ξ_y ≈ sin(ψ_bogie) × (v/Vc ratio)

Why Polach (not Kalker)?

Kalker's linear creep theory is only valid at very small slip (< 1%). Polach (2000) extends to full slip and correctly captures the non-linear saturation of the creep force toward µN at large slip. This is critical for traction/braking modelling near adhesion limit, especially during wheel-slip events.

8.3 Archard Wear Model

Archard wear volume: V_wear = K_wear × F_N × s / H where: K_wear = wear coefficient (dimensionless, material-pair specific) = 1×10⁻⁴ for R260 wheel / R260 rail at low slip = 3×10⁻⁴ at high slip (T-gamma > 15 N/mm) F_N = normal contact force [N] s = sliding distance [m] H = material hardness [Pa] = 260 × 9.81 × 10⁶ ≈ 2.55 GPa for R260 rail steel Sliding distance per km of travel: s = ξ × d [m slip per m travel] d = 1000 m (per km) Output: mm³ of material removed per km of travel per wheel

Parameter Effects on Wear:

  • Speed ↑ → increases through ξ (more lateral creep from hunting) and contact time changes
  • Conicity ↑ → more lateral hunting → ξ_y increases → wear rate increases rapidly near Vc
  • Axle load ↑ → F_N ↑ → both p₀ and wear volume increase proportionally
  • K_wear switch → changes at T-gamma = F_T × ξ threshold — detects severe wear regime

9. Ride Comfort & Suspension FRF

9.1 Suspension Frequency Response Function

Transmissibility (FRF) of secondary suspension: |H(f)| = √[(1 + (2ζr)²) / ((1 - r²)² + (2ζr)²)] where: r = f / f_n (frequency ratio) f_n = (1/2π) × √(k_s / m_car) [Hz] (natural frequency) ζ = c / (2√(k_s × m_car)) (damping ratio) In decibels: |H(f)|_dB = 20 × log₁₀(|H(f)|) Interpretation: |H| < 0 dB: attenuation (isolation) — above √2 × f_n |H| > 0 dB: amplification — near resonance (f ≈ f_n) |H| at f_n = 1/(2ζ) for underdamped system

Design Implications:

  • Target f_n ≈ 1.0–1.5 Hz for secondary suspension (below human sensitivity peak at 4–8 Hz)
  • At 0.25 damping: resonance peak ≈ 2× amplification, good isolation above 2 Hz
  • Softer springs (lower k_s): better isolation but more body sway → coupling with lateral dynamics
  • Harder springs: limits sway but transmits vibration more → worse Sperling Wz on rough track
